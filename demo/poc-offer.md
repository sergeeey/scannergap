# BlindSpotSec — Paid POC Offer

## Offer

A focused blind-spot audit for one repository, service, or security-critical flow.

This is a founder-led expert review supported by the ScannerGap benchmark methodology.

---

## Scope

The first POC covers:

- one repository or service;
- 3-5 security-critical flows;
- existing SAST output if available;
- local ScannerGap detector-rule smoke test where approved;
- selected blind-spot classes;
- manual review of scanner misses;
- optional custom rule suggestions.

---

## Deliverables

The client receives:

1. Executive summary
2. Scanner coverage matrix
3. Blind-spot taxonomy
4. 3-5 concrete examples
5. Evidence-backed explanation of missed or partially covered classes
6. Practical recommendations
7. Optional custom Semgrep rule suggestions
8. Next-step security checklist

---

## What this is

This is:

- a scanner gap assessment;
- a blind-spot review;
- a security methodology audit;
- an evidence-backed report.

---

## What this is not

This is not:

- a full penetration test;
- a production SaaS deployment;
- a replacement for existing SAST tools;
- a guarantee of complete vulnerability discovery;
- a certified benchmark of every scanner.

---

## Price hypothesis

First pilot price:

- $500-1000 for a scoped first audit.

This is a hypothesis until validated through customer conversations.

---

## Timeline

Typical pilot timeline:

- Day 1: scope and access
- Day 2-3: scanner gap analysis
- Day 4: report draft
- Day 5: review call and final report

---

## Safe sales wording

"I can run a focused blind-spot audit on one repo or service and show where your current scanner stack may be missing vulnerability classes. The output is a short evidence-backed report, not a SaaS deployment."

---

## Qualification questions

1. Which SAST tools do you currently use?
2. Do you have recurring vulnerability classes that scanners miss?
3. Do security findings often appear late in code review or pentest?
4. Which language/framework is most critical?
5. Would a short scanner coverage report be useful for your team?

---

## Success criteria

The POC is successful if the client receives at least one of:

- a previously unknown scanner blind spot;
- a clearer scanner coverage map;
- a custom detection rule idea;
- a prioritized review checklist;
- evidence that current scanner coverage is better than expected.

---

## Follow-up offer

If the first POC is useful:

- expanded audit across more repositories;
- custom rule development;
- recurring scanner-gap review;
- security pipeline advisory;
- fractional AI-security / AppSec consulting.
