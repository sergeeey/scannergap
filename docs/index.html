Many logical vulnerabilities require semantic or cross-function review beyond baseline pattern checks.

The 4 Types of Blind Spots

Type I: Cross-Function Taint

Scanner tracks data flow within one function but loses it across calls.

60-80% blind rate

Type II: Semantic Blindness

Security check exists in function A, but function B has none. Scanner assumes safe.

56-78% blind rate

Type III: Unknown Sinks

Scanner flags eval() but not boto3(endpoint=...)

100% blind rate

Type IV: Partial Bypass

SSRF filter exists on /import endpoint but not on /roles endpoint.

Case-dependent
